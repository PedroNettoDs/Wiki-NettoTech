Propagação do meu IP Público Dinâmico

Prefácio

Tenho interesse de acessar meu computador da onde eu tiver, seja via RDP, Apache Guacamole, FTP, MiniDLNA e entre outras formas.

Porém não estava disposto a pagar um IP PÚBLICO FIXO com a minha provedora, logo eu tinha idealizado fazer alguma ferramenta que a cada 1 dia, colhe-se o meu IP Público e altera-se no Cloudflare, com o meu agent de IA rodando, logo fiz esse questionamento e ele preparou esse 'Guia de Distribuição de IP público Dinâmico'

Então segue quais foram de fatos os comandos que funcionaram para mim

Duck Dns

Caso você não possua um dominio, orientamos acessar o DuckDNS: https://www.duckdns.org/index.jsp

Temos aqui a demonstração de um UX bem simples, conectei minha conta do GitHub e assim ele gerou para mim meu 'token'


Agora logado, a tela muda e concede acesso a seu 'token', assim como permite você descrever o seu Sub Dominio '.duckdns.org' e aparentemente, foi bem tranquilo, pois para configurar, foi somente um script dentro de Cron que rodaria de tempos em tempos via:

crontab -e

echo url="https://www.duckdns.org/update?domains={#Seu subdominio duck}&token={#Seu token}&ip=" | curl -k -o ~/duckdns/duck.log -K -

Só que eu não gostaria de ter um 'Sub Dominio' e sim um 'Dominio', exemplo:
Para eu acessar, seria 'nettotech.duckdns.org'. sendo que eu gostaria de ser 'nettotech.com.br', então prossegui com meu próximo candidato:

Colhendo token Cloudflare:

Ao acessar:
https://dash.cloudflare.com/profile/api-tokens
Eles já pontuam que é melhor adicionar 'Tokens de API da Conta'
https://dash.cloudflare.com/?to=/:account/api-tokens

Crie um token em 'Criar token'> e dentre os 'Modelos de tokens de API' já existe 'Editar DNS de zona':

Importante configurar Permissões:

Assim configurando 'Zona DNS para Editar e Lido', prosseguindo com o botão 'Continuar até o resumo', assim aparecerá o TOKEN, copie e salve ele para utilizar na próxima etapa.

e caso queira que o direcionado seja feito a algum subdominio, recomendamos a criação no momento

ddclient

Bom, dei uma pesquisada e compreendi, que esse programa foi projetado para atualizar automaticamente registros de DNS Dinâmico (DDNS), ou seja, se o seu IP de rede mudar, ele registra a mudança, maravilha, é justamente o que eu preciso.

Até mesmo a Cloudflare tem documentação sobre ele

Para instalação:

sudo apt install ddclient

Configurar o DDClient para seu domínio:

Assim que você der o sudo apt install, apareça a tela de configuração de pacotes:

Antes dessa tela, será solicitado para informar qual é o protocolo que será utilizado, no meu caso, coloquei 'cloudflare' e assim foi pedido o usuário e token.

Entretanto, vou recomendar que você dê Enter em tudo para fechar e configure manualmente o arquivo: /etc/ddclient.conf

# Configuration file for ddclient generated by debconf  
#  
# /etc/ddclient.conf  
  
protocol=cloudflare  
zone=seudominio.com.br
ttl=1  
login=token  
password=seu_token_de_api_conta  
use=web  
web=checkip.amazonaws.com  
sub.dominio.com.br

Configurado o arquivo, podemos rodar:

sudo systemctl enable ddclient
sudo systemctl restart ddclient
journalctl -u ddclient -f

Assim se ver alguma mensagem: SUCCESS:  updating,
